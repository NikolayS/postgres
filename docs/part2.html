№01

Inferred employer

commit-trailer emails · 2025 · fig 1

Robert Haas's dataset excludes employer. This section parses Author: and Co-Authored-By: trailers from 2025 commit messages on master, dedupes by email, and attributes each contributor to an employer. 300 distinct people, 1,825 credited contributions.

Method & limits. Employer is assigned in two stages: (1) the email domain, when it maps unambiguously to a company (fujitsu.com, postgrespro.ru, enterprisedb.com, etc.); (2) for personal-provider addresses (gmail.com, etc.), a manual map of the top ~30 contributors to their publicly-known employer. Contributors on personal email without a known mapping are bucketed as "unmapped / personal" — 544 / 1,825 = 29.8% of credited contributions fall in this bucket. Employment changes; some rows will be wrong; multi-author commits still credit only the first trailer. This is a proxy, not a leaderboard.

Back-branch activity

commits per release branch · 2025 · fig 3

Commits in 2025 per committer × branch. The master column is original commits; the REL_*_STABLE columns count only SHAs not reachable from master — i.e. true cherry-picks onto the back-branch. REL_18_STABLE branched off master on 2025-06-29, so its column shows post-branch work (bug-fix only) — 497 commits, not 1,930 as in a naive count.

x: release branch (newest → oldest) · y: committer · darker = more commits

Commitfest outcomes

patch status at CF close · 2025-Q1 2026 · fig 4

Patches queued for review pass through one-month commitfests. Seven ran from 2025-01 through PG19-Final (closed 2026-04-09). Each vertical bar is one CF; segments are the status at close of that CF, scraped from commitfest.postgresql.org. Caveat: a single patch can traverse several CFs before committing or being withdrawn; "Moved to different CF" here is per-CF exit, not distinct patches. The same patch can appear in multiple columns.

№06

Known blind spots

what part III would add
thread reconstruction

mbox parse of the full pgsql-hackers archive. Who starts threads, mean reply depth, median response latency per author. Expensive to crawl.

commitfest reviewer graph

Who reviews whom, with inferred reciprocity. The CF app has structured reviewer assignments per patch — not scraped here.

affiliation history

Same person, different employer, different year. Current mapping flattens to 2025 snapshot.

non-backend contributions

Drivers (JDBC, libpq alts), tooling (pgbouncer, PgBadger), extensions (TimescaleDB, Citus) — none in this dataset.

women & first-time contributors

The project's diversity signal is nowhere in the commit log. Needs surveys or self-identification.

tenure × throughput over time

How long does a new contributor take to become a committer? We have the endpoints but not the arc.

sources

Git master + REL_13..18_STABLE, shallow-cloned via GitHub mirror. Commitfest state pages at commitfest.postgresql.org. pgsql-hackers post counts from Robert Haas's hackers_emails_2025 table.

method

Author emails extracted from Author: / Co-Authored-By: trailers in commit bodies. Subdirectory churn from git log --numstat. Commitfest patches parsed from per-CF HTML status badges.
